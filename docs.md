# ScholarLink Backend API Documentation

Welcome to the **ScholarLink** API documentation. This backend provides a powerful hybrid engine for discovering academic journals, verifying PDF availability, and generating AI-driven summaries.

## Base URL
Default: `http://localhost:3001`

---

## 1. Search Journals
Discover academic publications from multiple sources with real-time PDF verification and AI analysis.

### Endpoint
`POST /api/searchjurnal`

### Request Headers
| Header | Value |
| :--- | :--- |
| `Content-Type` | `application/json` |

### Request Body
| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `query` | `string` | **Yes** | The search term, topic, or DOI. |
| `stream` | `boolean` | No | Set to `false` (Currently standard JSON response is recommended). |

**Example Request:**
```json
{
  "query": "Deep Learning in Medical Imaging"
}
```

### Response Body
The API returns a structured JSON object containing verified results and an AI-generated summary.

| Field | Type | Description |
| :--- | :--- | :--- |
| `query` | `string` | The original search query. |
| `status` | `string` | Execution status (e.g., "success"). |
| `results` | `array` | List of journal objects (see below). |
| `ai_summary` | `string` | A 3-4 sentence synthesis generated by NVIDIA GLM-4. |

**Journal Object Structure:**
```json
{
  "source": "OpenAlex | Google",
  "title": "String",
  "authors": "String",
  "year": "String (YYYY)",
  "doi": "String (URL)",
  "pdf_links": ["Array of direct PDF URLs"],
  "summary": "Full abstract or snippet"
}
```

## 2. Technical Implementation Details

### Hybrid Search Architecture
The backend executes search tasks in **parallel** using Go Routines for maximum performance:
1.  **OpenAlex API**: Fetches metadata, DOI, and official Open Access PDF links.
2.  **Google Search (CSE)**: Fallback for finding landing pages of PDFs.
3.  **Real-time Scraping**: For every result found, the system visits the landing page/DOI to extract `citation_pdf_url` from meta tags.
4.  **OJS Normalization**: Automatically converts "forced download" links (Open Journal Systems) into "browser-friendly view" links.

### AI Processing
*   **Model**: NVIDIA GLM-4.7 (via NVIDIA Build API).
*   **Context**: The AI receives a reconstructed abstract (from OpenAlex Inverted Index) or search snippets to generate its summary.

## 3. Environment Variables (.env)
To run this backend, ensure the following keys are present in your `.env` file:

```bash
# Google Custom Search (Optional but recommended)
SEARCH_ENGINE_KEY=your_google_api_key
SEARCH_ENGINE_ID=your_cse_id

# AI Summarization (Required for ai_summary)
NVIDIA_API_KEY=nvapi-xxxxxxxxxxxx
```

## 4. Development
To start the server:
```bash
go run src/*.go
```
The server will run on port `3001` by default.
